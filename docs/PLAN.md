# PLAN.md

프로젝트의 기획·방향·요구사항을 작성한다.

---

## 프로젝트 한 줄 요약

- 개발자 전용 대시보드(Dev Console)가 내장되어, 상태 변동과 비동기 데이터 흐름을 실시간으로 확인하고 테스트할 수 있는 프론트엔드 베이스 템플릿.

## 목표

- **구조적 시각화**: React 코드를 모르는 비개발자나 동료 개발자도 파일의 역할, 의존 관계, 데이터 수발신 구조를 2D 토폴로지 맵과 스펙 명세서를 통해 쉽게 이해하도록 지원.
- **실시간 흐름 추적**: 네트워크 API 요청/응답(및 중단), 전역 상태 변동 내역(State Diff), 커스텀 디버그 이벤트를 실시간 로깅 모니터로 수집 및 시각화.
- **테스트 시뮬레이션**: 네트워크 레이턴시(1.5초 지연) 및 통신 강제 실패(500 에러) 환경을 손쉽게 버튼 하나로 토글하여 예외 처리 메커니즘을 테스트.
- **완전 격리 번들링**: 메인 애플리케이션 번들과 개발용 대시보드(`dev.html` 진입점)를 완전히 분리하여 프로덕션 배포 시 보안 문제와 리소스 낭비를 차단.
- **문서화 통합**: 코드 저장소 내부의 마크다운 가이드북을 직접 파싱해 대시보드 내부에서 읽을 수 있는 뷰어 연동.

## 비목표 (하지 않을 것)

- 외부 거대 다이어그램 라이브러리(d3.js, React Flow 등)의 과도한 사용을 배제하여 번들 크기 최소화 및 컴포넌트 커스터마이징 유연성 확보.
- 프로덕션 배포 시 하단 플로팅 배너 및 콘솔 이동 링크 노출 금지.

## 대상 사용자 / 사용 맥락

- **기획자 / 비개발 직군**: 리액트 코드 문법 없이도 파일별 한글 설명 및 데이터 입출력 선을 보고 논리 구조를 검증하고 이해할 수 있음.
- **개발자**: 실시간으로 상태가 어떻게 변경되는지(이전/이후 차이값), API가 어떤 순서로 동작하고 언제 취소되는지 시각적으로 즉각 디버깅함.

## 요구사항·제약

- **기능**:
  - `dev.html` 및 `index.html` 다중 진입점(MPA) 빌드 지원.
  - 전역 스토어 상태 변경 시 `flowTracker`로 이전/이후 스냅샷의 얕은 비교 차이값 전달 및 로깅.
  - `api.ts` 래퍼 내부에 딜레이 처리 및 인위적 에러 강제 발생 미들웨어 로직 이식.
  - 2D 드래그 노드 좌표 및 파일별 주석 메모(학습 노트) `localStorage` 캐싱 연동.
- **비기능(성능, 접근성, 브라우저 등)**:
  - React 19의 외부 스토어 연동 훅(`useSyncExternalStore`)을 활용해 React 렌더링 성능 최적화.
  - ESLint 및 Prettier 포맷의 오류가 없어야 빌드 성공.
- **외부 연동·정책**:
  - JSONPlaceholder 등 검증용 가짜 API와 통신하여 상태/비동기 흐름 검증.
