STRUCTURE의 식별자(ID) 기준으로 요소별 기능·동작을 작성한다.

## 기능 목록

### 1. 공통 / 레이아웃

| ID (STRUCTURE와 동일)          | 트리거               | 동작·결과                                                              |
| ------------------------------ | -------------------- | ---------------------------------------------------------------------- |
| `floating-dev-console-trigger` | 배너 클릭            | 브라우저 새 탭에서 `/dev.html` 경로로 창을 띄움 (디버그 대시보드 진입) |
| `header-main`                  | 네비게이션 링크 클릭 | React Router에 의해 주소 이동 (`/`, `/about`)                          |

### 3. 개발자 콘솔 전용 (dev.html)

#### `tab-topology-map` (아키텍처 노드 맵)

| ID                       | 트리거                   | 동작·결과                                                                                                                  |
| ------------------------ | ------------------------ | -------------------------------------------------------------------------------------------------------------------------- |
| `canvas-control-panel`   | `배치도 초기화` 클릭     | 로컬 스토리지에 캐싱된 노드 좌표 삭제 -> 모든 노드 좌표를 `architectureDb.ts`에 정의된 `defaultPos` 초기값으로 복구        |
| `node-cards-layer`       | 노드 카드 마우스 드래그  | 드래그 실시간 캔버스 내 좌표 상태 갱신 -> 마우스 업(Drop) 시 `localStorage`의 `node_pos_<path>` 키값으로 최종 위치 캐싱    |
| `node-cards-layer`       | 노드 카드 마우스 호버    | 호버 상태로 지정된 노드와 연결된 다른 노드들 간의 SVG 베지에 곡선을 보라색 및 실시간 흐르는 애니메이션 점선으로 하이라이트 |
| `node-cards-layer`       | 노드 카드 좌클릭         | 클릭 대상 파일 노드를 선택 상태로 반영 -> 화면 하단의 `drawer-inspector-panel`을 오픈                                      |
| `drawer-inspector-panel` | 닫기 버튼(✕) 클릭        | 선택 상태 해제 -> 화면 하단 서랍 패널을 닫고 디폴트 메시지 영역으로 초기화                                                 |
| `inspector-spec-col3`    | 의존 파일 링크 클릭      | 대상 경로 `onNavigate(path)` 수행 -> 토폴로지 상의 대상 노드로 포커스 전환 및 서랍 내용 해당 파일 사양으로 갱신            |
| `inspector-spec-col3`    | 메모 텍스트에어리어 입력 | 입력 시 실시간 `localStorage`의 `node_memo_<path>`에 텍스트 저장 -> 상단에 "자동 저장됨..." 연녹색 깜빡임 알림 가동        |

#### `tab-flow-monitor` (실시간 데이터 흐름)

| ID                                   | 트리거                 | 동작·결과                                                                                                                        |
| ------------------------------------ | ---------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `monitor-control-bar` (필터)         | 필터 분류 클릭         | 선택된 분류(`api`, `state`, `event`, `all`)에 대응하는 로그 항목들만 필터링하여 출력                                             |
| `monitor-control-bar` (검색)         | 검색어 타이핑          | 로그 라벨 및 내부 JSON 내용 중 해당 검색어 문자열을 포함하는 로그만 필터링                                                       |
| `monitor-control-bar` (시뮬레이터🐌) | `지연 시뮬레이션` 토글 | `flowTracker.simulateLatency = checked` -> true 시 `api.ts` 내부 요청 시 1.5초 강제 딜레이 처리 및 `localStorage` 옵션 저장      |
| `monitor-control-bar` (시뮬레이터❌) | `강제 통신 실패` 토글  | `flowTracker.simulateError = checked` -> true 시 `api.ts` 내부 요청 시 무조건 500 ApiError 발생 처리 및 `localStorage` 옵션 저장 |
| `monitor-control-bar` (제어)         | `일시 정지` 클릭       | `isPaused` 상태를 true로 전환하여 트래커 브로드캐스트 리스너 작동 중단 (로그 수집 중지)                                          |
| `monitor-control-bar` (비우기)       | `로그 비우기` 클릭     | `flowTracker.clearLogs()` 실행 -> 로그 메모리 목록 배열 초기화 -> 화면 리스트 리셋                                               |
| `monitor-logs-list`                  | 로그 아이템 클릭       | 해당 로그를 디테일 대상으로 지정 -> 오른쪽 상세 인스펙터 패널에 데이터를 로딩                                                    |
| `monitor-json-viewer`                | `JSON 복사` 클릭       | `navigator.clipboard.writeText()` 호출 -> 클립보드에 인스펙터 상세 JSON 적재 -> 성공 알림 표출                                   |

#### `tab-docs-viewer` (설계 가이드북)

| ID             | 트리거         | 동작·결과                                                                                                                   |
| -------------- | -------------- | --------------------------------------------------------------------------------------------------------------------------- |
| `docs-sidebar` | 문서 단추 클릭 | 선택된 문서 경로로 `fetch(selectedDoc.path)` 시작 -> 로딩 중 표식 노출 -> 마크다운 파서 유틸리티 가동 -> HTML HTML로 렌더링 |
