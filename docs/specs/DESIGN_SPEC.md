# DESIGN_SPEC.md

## 목적

디자인 철학/토큰/컴포넌트 스타일 규칙/참고 코드 보관.

## 포함

- 철학, 키워드, 톤
- 색/타이포/간격 토큰
- 컴포넌트 상태별 스타일 규칙
- Stitch 등 참고 코드

## 제외

- 상태 소유/흐름
- 파일 작업 지도

## 갱신 트리거

- 토큰 변경
- 컴포넌트 스타일 규칙 변경

## 작성 규칙

- 토큰은 이름과 사용처를 함께 기록.
- 참고 코드는 출처/버전/날짜를 함께 남김.

## 권장 예문

- `color-primary-600`: primary 버튼 기본 배경
- `text-sm`: 본문 보조 텍스트
