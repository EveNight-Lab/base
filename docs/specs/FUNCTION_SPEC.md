# FUNCTION_SPEC.md

## 목적

STRUCTURE 식별자 기준 요소별 기능/동작 정의.

## 포함

- 트리거(클릭/제출/변경)
- 동작 결과(이동/API/상태 변경)

## 제외

- 레이아웃 비율
- 시각 스타일 규칙

## 갱신 트리거

- 기능 변경
- 동작 결과 변경

## 작성 규칙

- 한 항목은 `트리거 -> 결과` 형태로 짧게 작성.
- API는 메서드/엔드포인트를 함께 표기.

## 권장 예문

- ID: `header-search-submit`
- 트리거: `검색 버튼 클릭`
- 동작·결과: `GET /api/items 호출 -> 목록 갱신 -> 스크롤 top`
