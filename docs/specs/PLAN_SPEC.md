# PLAN_SPEC.md

## 목적

기획/목표/범위/요구사항 정의.

## 포함

- 프로젝트 한 줄 요약
- 목표, 비목표
- 대상 사용자/맥락
- 요구사항/제약

## 제외

- UI 비율 상세
- 상태 파일 매핑

## 갱신 트리거

- 요구사항 변경
- 우선순위 변경

## 작성 규칙

- 요구사항은 검증 가능한 문장으로 작성한다.
- 모호한 단어(적당히, 빠르게) 대신 기준을 함께 적는다.

## 권장 예문

- 목표: "로그인 사용자가 3클릭 이내에 주문 내역을 찾을 수 있어야 한다."
- 비목표: "v1에서 관리자 대시보드는 제공하지 않는다."
